JSX & Props — Student Profile Component

Pass data into a component via props and render it as JSX

JSX Props Functional Component Component reuse
Concept: JSX is a syntax extension that looks like HTML inside JavaScript. Props (properties) are read-only inputs passed from a parent component to a child — similar to function arguments. The same component can render different content by receiving different props. 
// StudentCard.jsx — Reusable card component

// Functional component receives props as its argument
function StudentCard({ name, branch, cgpa, skills }) {
  return (
    <div className="card">
      <h3>{name}</h3>
      <p>Branch: {branch}</p>
      <p>CGPA: <strong>{cgpa}</strong></p>

      {/* .map() renders an array of skill badges */}
      <div>
        {skills.map((skill, i) => (
          <span key={i} className="badge">{skill}</span>
        ))}
      </div>
    </div>
  );
}

// App renders StudentCard multiple times with different props
function App() {
  return (
    <div>
      <StudentCard
        name="Rahul Sharma"
        branch="CSE"
        cgpa={8.7}
        skills={["HTML", "CSS", "React"]}
      />
      <StudentCard
        name="Priya Patel"
        branch="ECE"
        cgpa={9.1}
        skills={["JavaScript", "Figma"]}
      />
    </div>
  );
}

useState Hook — Workout Session Counter

Manage component state that re-renders the UI on change

useState Hooks State Management Event Handling Conditional Rendering
Concept: useState is a React Hook that adds state to a functional component. It returns [value, setter]. Calling the setter triggers a re-render with the new value. Conditional rendering uses a ternary ? : to show different JSX based on state. 
import { useState } from 'react';

function WorkoutCounter() {

  // useState returns [currentValue, setterFunction]
  // Re-renders component every time setCount is called
  const [count,    setCount]    = useState(0);
  const [calories, setCalories] = useState(0);

  const addSession = () => {
    setCount(count + 1);
    setCalories(calories + 350); // ~350 kcal per session
  };

  const reset = () => {
    setCount(0);
    setCalories(0);
  };

  return (
    <div>
      <h2>Workout Sessions: {count}</h2>
      <p>Total Calories Burned: {calories} kcal</p>

      {/* Conditional rendering with ternary operator */}
      {count >= 5
        ? <p style={{color:'green'}}>🏆 Weekly goal reached!</p>
        : <p>{5 - count} more session(s) to reach goal</p>
      }

      <button onClick={addSession}>+ Log Session</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
}
// After clicking "+ Log Session" 3 times:
Workout Sessions: 3
Total Calories Burned: 1050 kcal
2 more session(s) to reach goal

// After 2 more clicks (total 5):
Workout Sessions: 5
Total Calories Burned: 1750 kcal
🏆 Weekly goal reached!

useEffect & Controlled Forms — Member Registration

Side effects, controlled inputs, and form validation in React

useEffect Controlled Form useState (multiple) onChange handler Form validation Side effects
Concept: useEffect runs code after render — used for side effects like updating the document title or fetching data. A controlled form ties every input's value to a state variable via value={state} and onChange, so React is the single source of truth for form data. 
import { useState, useEffect } from 'react';

function RegistrationForm() {

  // One state variable per form field (controlled form)
  const [name,  setName]  = useState('');
  const [email, setEmail] = useState('');
  const [plan,  setPlan]  = useState('Basic');
  const [error, setError] = useState('');
  const [submitted, setSubmitted] = useState(false);

  // useEffect: runs whenever 'name' changes
  // Updates document title as a side effect
  useEffect(() => {
    document.title = name
      ? `Registering: ${name}`
      : 'FitLife Registration';
  }, [name]); // dependency array — re-runs when name changes

  // useEffect with empty [] — runs once on mount (like componentDidMount)
  useEffect(() => {
    console.log('Form component mounted');
  }, []);

  const handleSubmit = (e) => {
    e.preventDefault();
    // Validation using if-else
    if (!name.trim()) {
      setError('Name is required.');
      return;
    }
    if (!email.includes('@')) {
      setError('Enter a valid email.');
      return;
    }
    setError('');
    setSubmitted(true);
  };

  // Conditional rendering: show success or form
  if (submitted) {
    return (
      <div>
        <h3>✅ Welcome, {name}!</h3>
        <p>Plan: {plan} | Email: {email}</p>
      </div>
    );
  }

  return (
    <form onSubmit={handleSubmit}>

      {/* Controlled input: value tied to state, onChange updates it */}
      <input
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Full Name"
      />

      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Email"
      />

      <select value={plan} onChange={(e) => setPlan(e.target.value)}>
        <option>Basic</option>
        <option>Pro</option>
        <option>Elite</option>
      </select>

      {error && <p style={{color:'red'}}>{error}</p>}
      <button type="submit">Register</button>
    </form>
  );
}
// When user types "Rahul" in Name field:
document.title → "Registering: Rahul" (useEffect side effect)

// On submit with valid data:
✅ Welcome, Rahul!
Plan: Pro | Email: rahul@example.com

// On submit with empty name:
❌ Name is required. (validation error shown inline)
